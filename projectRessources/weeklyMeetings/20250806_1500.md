## zu branch websocket_communication
- zusätzlich zu dem Werter sender, Type, arguments zusätzlich den Nutzer Token als Argument übersenden 8wäre hier sehr overkill, aber so nochmal sicherheit eingebudnen)
- somit wäre auch abgefnagen, dass andere SuS nicht für andere SuS etwa schreiben, da jeweils Instanz Token und Nutzer Token eine identische Signatur hat
- Broadcast Adresse sollte eigentlcih immer auf der ``x.x.x.255`` laufen, damit entsprehcend der broadcast empfangen werden kann (die letzte ADresse im Netzwerk, damit an alle geschickt werden kann)
- also für aktive Verbindung nicht die erste Adresse (``0.0.0.0`` "Netzwerkbasismaske") und die letzte Adresse verwenden
- => ergo hier Broadcast und aktive Verbindung 

- **WICHTIG:** die Broadcast Direktionalität umkehren => jeder Client würde über die ``.255`` als Broadcast den Server anfragen => dieser antwortet dann den anfragenden CLient auf der ipv4 Adresse von dem Client 
- **alternativ** wenn Test gestartet wird, dann hört broadcast auf **!!!Aber, da Netzwerk wohl fragil ist, ist dies eher zu vermeiden!!!**

- Hinweis: der Server kann von dem CLient die Subnetzmaske auslesen, sodass der Server nur in die SUbnetzmaske von dem Client die NAricht geschickt werden
